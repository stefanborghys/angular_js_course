Node.js

Introduction

Node.js is a JavaScript runtime, built on Chrome's V8 JavaScript engine.
Node.js ships with it's package ecosystem npm, the largest ecosystem of open source libraries in the world.

Runtime system
Software designed to support the execution of computer programs.
Runtime library
A program library designed to implement functions built into a programming language.
Chrome V8
V8 is Google's open source high-performance JavaScript engine, written in C++ and used in Google Chrome, the open source browser from Google.
It implements ECMAScript as specified in ECMA-262, 3rd edition, and runs on Windows XP or later, Mac OS X 10.5+, and Linux systems that use IA-32, ARM or MIPS processors.
V8 can run standalone, or can be embedded into any C++ application.

Installation

  1. download Node.js (v 4.2.3): Mac OS X (x64), Windows (x64), Linux (x64)
  2. add /usr/local/bin in your $PATH
  3. check if terminal / console command node -v returns the installed version number 'v4.2.3'
  4. check if terminal / console command npm -v returns the installed version number '2.14.7'

npm

Introduction

Package manager for JavaScript applications.
Comparable to Apache Maven dependency management in Java or Nuget for the Microsoft development platform, including .NET.
npm allows to install packages / dependencies, manage their versions and even deploy your own.

Update

Node comes with npm installed so you should have a version of npm.
However, npm gets updated more frequently than Node does, so you'll want to make sure it's the latest version.

  1. update npm via terminal / console command sudo npm install npm -g
  2. check if terminal / console command npm -v returns the installed version number '3.5.2'

Commands Overview

npm provides great documentation about it's environment and commands.
This section provides an overview of the most important and explains some concepts used by npm.
For more detailed information consult the https://docs.npmjs.com/ website.

Create a project

npm uses a package.json file to store and manage locally installed packages.
A package.json file can be compared to Apache Maven's .pom file.
Benefits:

  • documents packages your project depends
  • allows to specify the versions of a package that your project can use
  • makes build reproducable, meaning that its way easier to share with other developers

source: Using package.json

npm init
Creates a package.json file guided by a command line questionaire.
This can be compared to an Apache Maven .pom file
npm init --yes
Creates a default package.json file.

Install packages

Packages can be installed locally and globally, why this distinction?

Local packages are used and required by your web project to run when hosted.
Frameworks like Angular JS, Bootstrap, JQuery, ... and all many many many other libraries you use to style, implement, ... your web application are typical examples.
This is because they are directly required by your web application and get downloaded from the server when someone views your website.

Global packages are more used to support the development of your web application.
Like building it with Bower, running it using HTTP Server, adding tasks using Grunt, ...
As you can guess these dependencies don't have to be included when hosting the application.

Where to find packages?
Most projects who provide an install via npm will list it on their website, but you can always search for dependencies on the npm website.

npm install
If you have a package.json file in your directory then npm will look at the dependencies that are listed and download the latest versions using semantic versioning rules.
This is basically the only command necessary when you check out an existing project containing a package.json file.
npm install <package_name>
Installs a package locally.
Use this for packages you need inside your web project, like: Bootstrap, JQuery, Angular JS, ...
npm install <package_name> --save
Installs a package locally and adds it to package.json dependencies.
npm install <package_name> --save-dev
Installs a package locally and adds it to package.json development dependencies.
This can be handy when you want to experiment with certain dependencies during the development cycle.
npm install -g <package_name>
Installs a package globally.
Use this for packages required by tools or (JavaScript) applications that process your project, like Grunt, HTTP Server, ...
This command also updates a specific global package in contrast to npm update -g.

Semantic versioning
Semantic versioning is a standard that a lot of projects use to communicate what kinds of changes are in this release.
It's important to communicate what kinds of changes are in a release because sometimes those changes will break the code that depends on the package.

Update packages

npm outdated
Check for outdated local packages.
npm outdated -g --depth=0
Check for outdated global packages.
npm update
Update ALL local packages by running this command in the same directory as your package.json file.
npm update -g
Update ALL global packages.
npm install -g <package_name>
Update a specific global package.

Remove packages

npm uninstall <package_name>
Remove a local package from your node_modules directory.
npm uninstall --save <package_name>
Remove a local package from the dependencies in package.json
npm uninstall -g <package_name>
Remove a global package.

Bootstrap

Introduction

Bootstrap is the most popular HTML 5, CSS 3 and JavaScript framework for quickly developing responsive, mobile first projects on the web.

What can you do with Bootstrap?
It allows to quickly prototype or develop web applications by using pre defined components which are build out of standard HTML 5 tags and sets of pre defined CSS 3 styles.
This provides responsive design out of the box, which is extendable and adapted by a large group of people, projects, frameworks, ...

Responsive web design
Responsive web design ( RWD) is an approach to web design aimed at crafting sites to provide an optimal viewing and interaction experience
easy reading and navigation with a minimum of resizing, panning, and scrolling across a wide range of devices (from desktop computer monitors to mobile phones).

  • open source
  • responsive
  • default HTML 5, CSS 3, JavaScript components
  • extendable (themes: Bootswatch, Wrap Bootstrap, ...)
  • great adaptation (UI Bootstrap, Bootstrap for Drupal, ...)
  • JQuery dependency necessary for Bootstrap JavaScript plugins
